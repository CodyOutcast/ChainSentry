# ChainSentry 用户使用说明

## 1. 这个系统能帮你做什么

ChainSentry 用来在你签名一笔链上交易之前，先告诉你：

- 这笔交易实际在干什么
- 它为什么可能有风险
- 签了以后可能出什么后果
- 你现在更适合继续、再检查一下，还是直接拒绝

它当前主要识别三类风险：

- 大额或无限授权
- 与可疑目标地址交互
- 签名后可能出现的异常资产流出或权限升级

## 2. 使用前准备

项目包含前端和后端两个部分，使用前需要把它们都启动起来。

### 安装依赖

如果你使用虚拟环境：

```bash
python3 -m venv .venv
.venv/bin/pip install -r backend/requirements.txt
cd frontend
npm install
cd ..
```

如果你已经在本机装好了依赖，也可以直接用 `python3` 代替 `.venv/bin/python`。

## 3. 启动系统

### 启动后端

推荐使用图模型版本：

```bash
CHAIN_SENTRY_PREDICTOR_BACKEND=graph-model \
PYTHONPATH=backend python3 -m uvicorn app.main:app --reload
```

如果你只想使用启发式版本：

```bash
CHAIN_SENTRY_PREDICTOR_BACKEND=heuristic-fallback \
PYTHONPATH=backend python3 -m uvicorn app.main:app --reload
```

后端地址：

- `http://localhost:8000`
- 健康检查：`http://localhost:8000/health`

### 启动前端

```bash
cd frontend
cp .env.example .env
npm run dev
```

前端地址：

- `http://localhost:5173`

## 4. 第一次使用时怎么操作

打开前端后，按下面步骤使用：

1. 进入页面后，先选择一个 demo 场景，或者连接钱包后手动填写交易信息。
2. 检查左侧表单中的交易字段。
3. 点击 `Analyze Transaction`。
4. 等待右侧生成风险报告。
5. 根据报告中的推荐动作决定是否继续签名。

如果你只是演示项目，建议先直接使用内置场景，不需要自己手输复杂交易。

## 5. 报告结果怎么看

分析完成后，页面会显示一张风险报告卡。

你重点看这几个部分：

### `overall severity`

表示整体风险等级：

- `low`：风险较低
- `medium`：建议再检查
- `high`：高风险
- `critical`：非常高风险

### `recommended action`

表示系统建议你怎么做：

- `proceed`：可以继续
- `inspect further`：先再核查
- `reject`：建议拒绝

### `Expected action`

这一项告诉你：**这笔交易实际准备做什么。**

比如：

- 给某个合约授权花你的代币
- 给某个 operator 全局权限
- 通过某个 router 进行 swap

### `Risk reason`

这一项告诉你：**为什么系统认为它危险。**

比如：

- 授权额度过大
- 目标地址在可疑名单中
- 模拟结果显示会出现超出预期的资产流动

### `Possible impact`

这一项告诉你：**如果你签了，最可能出什么事。**

比如：

- 对方之后可以继续扣你的币
- 资产可能被进一步转走
- 对方会获得持续权限

### `Recommended action`

这一项告诉你：**你下一步应该怎么做。**

比如：

- 如果不完全信任对方就拒绝
- 缩小授权额度
- 先核对目标地址和用途

### `Simulation`

这部分会显示系统推测出来的后续效果，用来补充钱包弹窗本身没说明白的后果。

### `Evidence`

这部分展示一些支撑线索，例如：

- 方法名
- approval 数值
- spender 地址
- 图模型分数
- 图结构大小

它是辅助信息；普通用户先看上面的四行解释就够了。

## 6. 推荐的演示顺序

如果你要给老师或同学演示，建议按这个顺序：

1. `Standard Transfer`
   说明系统不会把所有交易都判成高风险
2. `Unlimited Approval To Drain Contract`
   说明系统能识别无限授权
3. `Collection-Wide Operator Approval`
   说明系统能识别权限升级
4. `Swap Route With Hidden Outflow`
   说明系统能识别表面正常、实际有隐藏后果的交易

## 7. 常见问题

### 为什么有时还能看到 heuristic 的痕迹？

因为当前系统是 hybrid 方案：图模型和启发式规则一起工作。这样演示更稳，也更容易解释。

### 为什么项目不接真实风控 API？

因为课程项目已经主动收缩范围，目标是做一个小而清楚、能解释的 prototype，而不是生产级风控平台。

### 为什么模型不是直接判断所有链上风险？

因为当前系统只聚焦几类典型、容易解释的风险场景，不追求全覆盖。

## 8. 一句话理解这个项目

ChainSentry 不是替你“看懂整个区块链”，而是帮你在签名前先看懂一笔交易到底在授权什么、可能哪里有坑、你下一步该怎么做。
