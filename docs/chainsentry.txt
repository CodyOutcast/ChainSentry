ChainSentry: An Explainable Pre-Signature Risk Analysis System for Blockchain Transactions
Strengths
This proposal captures a very real pain point in Web3: many users do not really know what they are authorizing or what will happen before they sign a transaction. The proposal goes one step further by turning “risk detection” into “explanations users can understand,” which is especially valuable. It does not stop at a developer-facing analysis tool, but tries to turn the system into something that genuinely supports user decision-making.
Weaknesses
The main thing to watch here is that the project’s ambition is rather large. Static analysis, dynamic simulation, graph learning, an explanation layer, and wallet integration together already approach the scale of a full product prototype. At the same time, since this project starts from the user side, success does not really depend on how many technical modules are added or how fancy the system looks. It depends more on whether the system is actually grounded in a good understanding of user behavior and how users interpret risk. Right now, the proposal looks more like a strong design direction than a well-developed account of how users understand risk, receive explanations, or decide which information is truly useful.
Suggestions
If the project continues based on the current proposal, the technical focus during implementation should be placed more on understanding users, rather than continuing to add more technical modules. The team should think first about what users are most confused about before signing, what kinds of prompts they notice, what level of explanation is helpful, and what level becomes too much information. Even if the final system is not very large and only handles a few typical risks and explanation styles, that would still be more convincing than a broad security platform that may not actually improve user decisions. If necessary, some of the originally proposed technical modules could also be reduced.
